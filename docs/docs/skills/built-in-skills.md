# 内置技能详情

所有内置技能按 `config.json` 中 `tasks.enabled` 的顺序在同一浏览器会话中依次执行。默认顺序如下：

## welkin-moon — 月卡每日领取

- **ID**: `welkin-moon`
- **说明**: 登录 → 启动游戏 → 领取月卡奖励
- **超时**: 10 分钟（`600000ms`）
- **重试**: 1 次

### 执行顺序

**必须排在第一个。** 此技能负责打开云原神页面、完成登录流程、启动游戏。后续所有技能都依赖它建立的游戏内会话。

### 前置条件

- 已购买"空月祝福"（月卡）。如果未购买，技能会在找不到领取按钮时报告完成
- 已通过 `--no-headless` 完成过至少一次手动登录，cookie 已保存

### 已知限制

- 云原神可能出现排队等待，技能会自动等待排队完成
- 启动后可能弹出新手引导、用户协议等弹窗，技能会尝试关闭
- 如果游戏画面长时间无响应（如 canvas 未激活），可能需要点击激活
- 网络波动可能导致启动超时，建议在网络稳定时运行

## claim-mail — 邮件领取

- **ID**: `claim-mail`
- **说明**: 打开邮箱 → 一键领取所有附件
- **超时**: 5 分钟（`300000ms`）
- **重试**: 1 次

### 执行顺序

需要在 `welkin-moon` 之后执行（游戏已启动，角色在主界面）。

### 前置条件

- 游戏已启动并进入主界面（由 `welkin-moon` 完成）

### 已知限制

- 如果邮件图标没有红点（无新邮件），技能会直接报告完成
- 邮件列表可能需要滚动，技能会尝试滚动查找
- 领取附件后可能弹出物品展示弹窗，技能会自动关闭
- 如果主界面有其他弹窗遮挡，技能会先尝试关闭弹窗

## expedition-collect — 探索派遣收取

- **ID**: `expedition-collect`
- **说明**: 冒险之证 → 收取已完成派遣 → 重新派遣
- **超时**: 10 分钟（`600000ms`）
- **重试**: 1 次

### 执行顺序

需要在 `welkin-moon` 之后执行。与 `claim-mail` 无依赖，但建议按默认顺序排列。

### 前置条件

- 游戏已启动并进入主界面

### 已知限制

- 通过 Paimon 菜单 → 冒险之证进入，需要先打开 Paimon 菜单
- 冒险之证可能默认打开其他标签页（如"委托"），需要切换到"探索派遣"标签
- 部分角色可能因为在队伍中而无法派遣，技能会跳过
- 收取后需要重新选择角色和派遣时长，默认选择 20 小时
- 物品领取弹窗需要关闭后才能继续操作

---

## battle-pass-claim — 纪行奖励领取

- **ID**: `battle-pass-claim`
- **说明**: 打开纪行 → 领取可领取的等级奖励
- **超时**: 5 分钟（`300000ms`）
- **重试**: 1 次

### 执行顺序

建议排在最后执行。需要在 `welkin-moon` 之后。

### 前置条件

- 游戏已启动并进入主界面

### 已知限制

- 纪行可以通过 F5 快捷键或 Paimon 菜单打开，但云游戏中 F5 可能不生效，技能会回退到菜单方式
- 纪行界面有多个标签页（纪行任务、纪行之证等），需要确保在奖励领取页面
- 只领取已解锁（亮起）的奖励，灰色未解锁的会跳过
- 版本更新后纪行可能重置，界面布局可能变化
- 领取奖励后的物品展示弹窗需要关闭
